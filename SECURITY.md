# 安全政策

| 版本 | 支援狀態 |
|------|----------|

## 報告安全漏洞

如果你發現安全漏洞，請**不要**在公開的 Issue 中報告。

請將漏洞詳情發送至專案維護者。

## 安全最佳實踐

1. **永遠不要提交 .env 文件** — 已在 `.gitignore` 中排除
2. **使用強密碼** — 管理員密碼至少 16 字元
3. **定期更換 SECRET_KEY** — 生產環境中每 90 天更換一次
4. **限制 API 存取** — 交易所 API Key 設定 IP 白名單
5. **監控登入日誌** — 定期檢查 `login_log` 表中的異常登入
